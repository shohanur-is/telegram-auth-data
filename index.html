Hash Verification

Verification Steps

1
Create data-check-string

Concatenate all received fields (except hash) sorted alphabetically in the format key=<value> with line feed separator:

auth_date=<timestamp>\nfirst_name=<name>\nid=<user_id>\n...
2
Compute secret key

Compute SHA256 hash of your bot token:

secret_key = SHA256(bot_token)
3
Compute HMAC-SHA256 signature

Using the secret key, compute HMAC-SHA256 signature of the data-check-string:

computed_hash = HMAC_SHA256(data_check_string, secret_key)
4
Compare hashes

Compare the computed hash with the received hash value (case-insensitive):

if (computed_hash == received_hash) {
    // Data is authentic
} else {
    // Data has been tampered with
}

Official Documentation

Receiving Authorization Data

After successful authorization, Telegram provides data in two ways:

  1. Redirect to the URL specified in data-auth-url with parameters: id, first_name, last_name, username, photo_url, auth_date, and hash
  2. Callback function specified in data-onauth with a JSON object containing the same fields

Verifying Authorization

To verify the authentication and data integrity:

  1. Compare the received hash parameter with the hexadecimal representation of the HMAC-SHA-256 signature
  2. The signature is computed from the data-check-string using the SHA256 hash of the bot's token as the secret key

Data Check String Format

key1=<value1>\nkey2=<value2>\n...keyN=<valueN>

Where:

  • Keys are sorted alphabetically
  • Only include fields present in the received data (except hash)
  • Each key-value pair is separated by a newline character (\n, 0x0A)

Verification Process

data_check_string = ...
secret_key = SHA256(bot_token)
if (hex(HMAC_SHA256(data_check_string, secret_key)) == hash) {
    // Data is authentic
}

Checking Auth Date

To prevent use of outdated data, check the auth_date field which contains a Unix timestamp:

current_time = get_current_unix_timestamp()
if (current_time - auth_date > 86400) { // 24 hours
    // Data is too old
}
